Here's what you can test for Phase 3:

```bash
# Setup
cd /tmp && rm -rf vt && mkdir vt && cd vt
VRIT=/Users/dev0/sandbox/claude/mygit/target/debug/vrit
$VRIT init
echo 'user.name = You' >> .vrit/config
echo 'user.email = you@test.com' >> .vrit/config

# Base commit
echo "line one" > file.txt
echo "only in main" > main-only.txt
$VRIT add .
$VRIT commit -m "Initial"

# 1. Branch operations
$VRIT branch feature         # create
$VRIT branch                 # list — main has *, feature doesn't
$VRIT branch -d feature      # delete
$VRIT branch                 # only main
$VRIT branch -d main         # error: can't delete current branch

# 2. Checkout + file switching
$VRIT branch feature
$VRIT checkout feature
echo "feature work" > feature.txt
echo "changed by feature" > file.txt
$VRIT add .
$VRIT commit -m "Feature changes"
$VRIT checkout main
cat file.txt                 # should be "line one" again
ls feature.txt               # should NOT exist
$VRIT checkout feature
cat file.txt                 # should be "changed by feature"
ls feature.txt               # should exist

# 3. Dirty checkout refusal
$VRIT checkout main
echo "uncommitted" > file.txt
$VRIT checkout feature       # should REFUSE with clear message

# Reset for merge tests
$VRIT checkout -- file.txt   # restore from HEAD
$VRIT status                 # clean

# 4. Fast-forward merge
$VRIT checkout main
$VRIT merge feature          # FF — main was ancestor of feature
$VRIT log                    # no merge commit, just feature's commits
cat file.txt                 # "changed by feature"
ls feature.txt               # exists now

# 5. Diverging clean merge (three-way, no conflict)
$VRIT branch side
$VRIT checkout side
echo "side addition" > side.txt
$VRIT add side.txt
$VRIT commit -m "Side work"
$VRIT checkout main
echo "main addition" > main-new.txt
$VRIT add main-new.txt
$VRIT commit -m "Main work"
$VRIT merge side             # "Merge made by the 'recursive' strategy."
$VRIT log                    # merge commit with "Merge: xxx yyy"
ls side.txt main-new.txt     # both exist

# 6. Merge with conflict
$VRIT branch conflict
$VRIT checkout conflict
echo "conflict version" > file.txt
$VRIT add file.txt
$VRIT commit -m "Conflict change"
$VRIT checkout main
echo "main version" > file.txt
$VRIT add file.txt
$VRIT commit -m "Main change"
$VRIT merge conflict         # CONFLICT message
cat file.txt                 # should show <<<<<<< main / ======= / >>>>>>> conflict
$VRIT status                 # "You have unmerged changes"

# 7. Merge --abort
$VRIT merge --abort
cat file.txt                 # back to "main version"
$VRIT status                 # clean

# 8. Resolve conflict manually
$VRIT merge conflict
echo "resolved content" > file.txt
$VRIT add file.txt
$VRIT commit -m "Merge resolved"
$VRIT log                    # merge commit with two parents

# 9. Detached HEAD
$VRIT log                    # pick a SHA from output
$VRIT checkout <paste-sha>   # warning about detached HEAD
$VRIT status                 # "HEAD detached at xxxxxxx"
$VRIT checkout main          # back to normal

# 10. Already up to date
$VRIT merge main             # "Already up to date."

# 11. Merge refuses dirty tree
echo "dirty" > file.txt
$VRIT merge conflict         # should refuse: "commit or stash before merging"
```

Key things to validate:
- **File switching**: checkout actually changes files on disk, not just refs
- **Dirty protection**: both checkout and merge refuse when tracked files are dirty
- **Conflict markers**: `<<<<<<<`/`=======`/`>>>>>>>` have correct branch names
- **Merge abort**: fully restores pre-merge state
- **Conflict resolution**: `add` + `commit` after resolving creates a two-parent merge commit
- **Log shows merge parents**: "Merge: xxx yyy" line on merge commits
